# 参考資料置き場

## ルール
READMEを編集したら参考資料について書かれているissueで報告。
 ->　ジャンル変更や目的変更など
説明の粒度などは（今の所）意識しなくても良い。がURLだけはできるだけ避けたい。

## 目的
１．レビュー時に資料の提示をしやすくするため
２．実装時の補助
３．課題修了後に見返す

## ジャンル
- comprehensive
	- 総合的なこと。
	- シェルそのもの、複数事項に渡るもの
- perse
	- 構文解析、コマンド解析
	- シェルをコマンドインタプリタとして捉えた時に参考にしたもの
- system
	- シェルをカーネルとの橋渡し役として捉えた時に参考にしたもの
- develop
	- シェルそのものには関係しないが開発手法など、プロジェクト実行時に参考にしたもの。
